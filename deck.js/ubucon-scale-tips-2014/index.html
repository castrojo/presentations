Common Guidelines

  • You're not supposed to be stomping all over the disk!
  • Prefer dotfiles to messing with /etc
  • Read the warnings (I know right)

Dotfiles can also get you into trouble

  • .gconf - "Human Readable" XML
  • Easy to backup, so do it
  • You can generally recover from messing up in your home directory
  • Suspect something? Switch users and try it

Things to never tweak unless you know what you are doing

  • Plumbing - kernel, Xorg, init, grub
  • Power - you are dumber than the kernel
  • If it was an amazing tweak our kernel team would have shipped it. :)

PPA Guidelines

  • Do you trust that person on the internet?
  • Even if you do do you trust their commitment to that PPA?
  • Work towards not needing PPAs, either via backports or upgrades
  • Granular PPAs > one huge pile

PPA Guidelines cont.

  • We disable PPAs on upgrade, do an audit
  • Try to avoid "plumbing" PPAs if you are not an expert
  • The Hardware Enablement Stack can really help you here
  • Consider PPAing, and then removing it

The Hardware Enablement Stack

  • Comes with the LTS point releases.
    • sudo apt-get install linux-generic-lts-saucy linux-headers-generic-lts-saucy xserver-xorg-lts-saucy
  • Gets you to the next LTS, great for new hardware
